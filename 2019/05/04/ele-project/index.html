官方文档说明：

mounted：

el 被新创建的 vm.$el 替换，并挂载到实例上去之后调用该钩子。如果 root 实例挂载了一个文档内元素，当 mounted 被调用时 vm.$el 也在文档内。

mounted 不会承诺所有的子组件也都一起被挂载。如果希望等到整个视图都渲染完毕，可以用 vm.$nextTick 替换掉 mounted 。

VM.$nextTick() ：

VM.$nextTick(callback)将回调延迟到下次 DOM 更新循环之后执行。在修改数据之后立即使用它，然后等待 DOM 更新。

因此需要操作原生DOM对象，或者访问对象的一些属性时，最好是在该方法的回调中执行，这样可以保证DOM是已经根据最新的数据更新过的，不会出现访问不到的异常情况，而mounted并不能保证DOM是最新的。

16、better-scroll一些使用注意事项

  1. 引入better-scroll后需要显式地调用new BScroll(需要作用的DOM元素，配置对象)来进行初始化。

  2. 需要注意初始化的时机：因为better-scroll的作用原理是通过计算元素的高度来实现的，因此需要在DOM全部生成后才能进行初始化，否则不生效。在本项目中，由于需要作用的元素是需要先从服务器拿取数据后才动态生成DOM，因此不能在mounted钩子中执行初始化，需要在数据获取完成后通过调用VM.$nextTick(callback)函数，在回调中去执行初始化。

  3. better-scroll的一些可选配置项

    • click: true：better-scroll默认阻止了元素的touch事件，需通过该配置项设置开启。
    • probeType: 3：开启实时监听滚动的位置的状态

  4. better-scroll的一些方法

    • 可监听“scroll”事件：回调函数以实时滚动的位置作为参数，可以通过监听该事件，实时获取滚动位置。
    • better-scroll提供了一个滚动到指定元素位置的API：scrollToElement(需要滚动到指定位置的元 素, 滚动动画的时间毫秒)
